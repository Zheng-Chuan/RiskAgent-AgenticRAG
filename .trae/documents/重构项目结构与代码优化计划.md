# 重构计划：提升可读性与架构优化

根据您的需求和代码库分析，我制定了以下重构计划，旨在让代码更易读、结构更清晰、逻辑更内聚。

## 1. 移除不必要文件与代码清理 (Cleanup)
- **删除 `.pylintrc`**: 之前移除了 pylint，配置文件已无用。
- **删除 `sys.path` Hacks**: 移除 `gradio_app.py` 和 `demo_cli.py` 中手动修改 `sys.path` 的代码（通过规范的包导入解决）。

## 2. 统一配置管理 (Configuration Centralization)
**现状**: `os.getenv` 散落在 `gradio_app.py`, `milvus_index.py` 等多处，难以阅读和维护。
**改进**:
- 创建 `src/riskagent_rag/settings.py`。
- 定义 `Settings` 类，统一管理所有环境变量（Milvus, LLM, Embeddings 等）。
- 所有业务代码只从 `settings` 对象读取配置，不再直接调用 `os.getenv`。

## 3. RAG 模块职责拆分 (Refactor `milvus_index.py`)
**现状**: `milvus_index.py` 混合了文档切分、Embedding 模型加载、Milvus 连接管理等多个职责，代码臃肿。
**改进**:
- 拆分为三个专注的模块：
    - `src/riskagent_rag/rag/embeddings.py`: 专门处理 Embedding 模型加载与缓存。
    - `src/riskagent_rag/rag/vectorstore.py`: 专门处理 Milvus 连接与 CRUD。
    - `src/riskagent_rag/rag/ingestion.py`: 专门处理文档加载与切分逻辑。
- `milvus_index.py` 将被移除或仅保留作为入口兼容（如果是外部 API）。

## 4. 核心逻辑下沉与 Facade 模式 (Logic Encapsulation)
**现状**: `gradio_app.py` 和 `demo_cli.py` 包含大量重复的初始化逻辑（如 `_ensure_graph`）和状态管理。
**改进**:
- 创建 `src/riskagent_rag/app.py` (或 `system.py`)。
- 封装 `RiskAgentSystem` 类，提供统一的高层 API：
    - `initialize()`: 加载配置、连接数据库。
    - `chat(query, history)`: 处理对话核心流程。
    - `build_knowledge_base()`: 触发索引构建。
- `gradio_app.py` 和 `demo_cli.py` 瘦身，仅负责 UI/CLI 交互，直接调用 `RiskAgentSystem`。

## 5. 增强类型提示与中文文档 (Type Hints & Docs)
- 为所有新提取的函数和类添加 Python 类型提示。
- 按照 Google 风格添加清晰的中文 Docstrings，解释“为什么这样做”而不仅仅是“做了什么”。

## 执行顺序
1.  清理文件与 Hack。
2.  建立统一配置 `settings.py`。
3.  拆分 `milvus_index.py`。
4.  封装 `RiskAgentSystem` 并重构入口文件。
5.  全面测试回归。
