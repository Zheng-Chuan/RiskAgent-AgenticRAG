根本原因分析
================================================================================

类型1: 环境依赖问题 (影响 +5-8%)
  ├─ email-validator 版本要求 → Pydantic 2.x 无法初始化
  │  └─ 导致: api/server.py 无法导入 (7个测试)
  ├─ transformers 缺少 LRScheduler → 初始化失败
  │  └─ 导致: app.py 无法初始化 (6个测试)
  └─ pytest.mark.skipUnless 不存在 → 测试集合失败
     └─ 导致: 2个场景测试文件无法加载

类型2: 缺少测试实现 (影响 +20-30%)
  ├─ Evaluation 模块完全零测试
  │  └─ 14个文件, 1,743行代码
  ├─ API 端点缺少完整路径测试
  │  └─ 3个端点, 216行代码
  └─ CLI 命令无集成测试
     └─ 97行代码

类型3: Mock 和 Fixture 不足 (影响 +8-15%)
  ├─ RAG 混合检索需要:
  │  ├─ Milvus 向量存储 mock
  │  ├─ 嵌入模型 mock
  │  ├─ BM25 稀疏索引 mock
  │  └─ RRF 重排序逻辑
  ├─ LLM 生成需要:
  │  ├─ 路由选择 mock
  │  ├─ 重试逻辑
  │  ├─ 缓存命中路径
  │  └─ Token 计数
  └─ 评测需要:
     ├─ LLM 评估结果 mock
     └─ 指标计算结果

类型4: 架构约束 (影响 +3-5%)
  ├─ Docker 中间件依赖 (Milvus, Redis)
  ├─ LLM API 真实 credentials
  └─ 复杂的端到端集成
